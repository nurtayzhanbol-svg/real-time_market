# SQLAlchemy Playground

This folder is separate from the app code. Use it to practice SQLAlchemy concepts
with SQLite before wiring anything into the FastAPI/PostgreSQL project.

Run the starter script:

```bash
.venv/bin/python sqlalchemy_playground/playground.py
```

It creates `sqlalchemy_playground/playground.db`, inserts sample market data, and
prints a few query results.

## Mental Model

- SQL tables become Python classes.
- Table columns become class attributes.
- A `Session` is the object you use to talk to the database.
- `select(...)` builds SQL queries with Python expressions.
- `relationship(...)` lets linked rows behave like connected Python objects.
- `commit()` persists changes.

In the real app:

- Pydantic models describe API input/output.
- SQLAlchemy models describe database tables/rows.

## Exercises

### 1. Read the Models

Open `playground.py` and identify:

- Which class maps to the `instruments` table?
- Which class maps to the `price_events` table?
- Which column is the foreign key?
- Which relationship links an instrument to its price events?

Goal: be able to explain how `Instrument.price_events` connects to
`PriceEvent.instrument_id`.

### 2. Add Another Instrument

In `seed_data()`, add a new instrument:

```python
Instrument(ticker="NVDA", asset_type="stock", currency="USD")
```

Then add at least three `PriceEvent` rows for it.

Run the script and confirm it appears in the printed output.

### 3. Query One Ticker

Add a function named `print_history_for_ticker(session: Session, ticker: str)`.

It should:

- Find the instrument by ticker.
- Print every price event for that ticker ordered by `event_time`.
- Print a helpful message if the ticker does not exist.

Hint:

```python
select(Instrument).where(Instrument.ticker == ticker)
```

### 4. Latest Price Query

Add a function named `latest_price(session: Session, ticker: str)`.

It should return the newest `PriceEvent` for the ticker.

Hint:

```python
order_by(PriceEvent.event_time.desc())
```

### 5. Aggregate Query

Add a function that prints total traded volume per ticker.

Hint: SQLAlchemy can call SQL functions:

```python
from sqlalchemy import func
```

Then use:

```python
func.sum(PriceEvent.volume)
```

### 6. Update a Row

Write code that changes an instrument name:

```python
AAPL -> Apple Inc.
```

Then commit it and query it again to prove the change persisted.

### 7. Delete and Observe Cascade Behavior

Delete one `Instrument`.

Before committing, predict what should happen to its `PriceEvent` rows.

Then query `price_events` and check whether your prediction was right.

### 8. Cause a Constraint Error

Try inserting a `PriceEvent` with a negative price.

Observe the error, then rollback the session:

```python
session.rollback()
```

Goal: understand that database constraints still matter even when using ORM
classes.

### 9. Compare ORM to Raw SQL

Print the SQL generated by one query:

```python
print(statement)
```

Then write the equivalent raw SQL in this README under your notes.

### 10. Connect Back to the Real Project

Open `market_pipeline/models.py`.

Find the production versions of:

- `Instrument`
- `PriceEvent`
- `MarketMetric`
- `LatestMarketPrice`

Compare them to the playground models. The production models are more detailed,
but the core ideas are the same.
